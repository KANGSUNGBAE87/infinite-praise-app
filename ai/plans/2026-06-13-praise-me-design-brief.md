# 칭찬해줘 디자인 기획안 v0.1

작성일: 2026-06-13
상태: current-brief
기준 구현: 2026-06-08 디자인 MVP

## 한 줄 방향

`칭찬해줘`는 사용자가 지금 듣고 싶은 말을 5개 버튼 중 하나로 고르면,
홈 화면 안에서 바로 한마디가 나오고 로컬 오디오가 재생되는 따뜻한 오디오 셀프케어 앱이다.

디자인의 목표는 멋진 랜딩 페이지가 아니라,

> 사용자가 앱을 열자마자 "아, 여기서 버튼 하나 누르면 지금 필요한 말을 들을 수 있구나"를 이해하는 것

이다.

## 디자인 키워드

- 따뜻함
- 즉시성
- 다정함
- 가벼운 안정감
- 토스식 정보 구조
- 모바일 앱스토어에 어색하지 않은 완성도
- 유치하지 않은 자존감 앱

## 화면 범위

MVP는 홈 단일 화면으로 간다.

새로운 재생 전용 화면을 만들지 않고, 홈 안에서 현재 한마디 카드가 갱신된다.

## 추가 UX 검토: 직접 선택 흐름

상위 버튼을 눌렀을 때 바로 다음 페이지로 이동해 5개 오디오 중 하나를 고르게 하는 방식은
v0.1 메인 흐름으로 적용하지 않는다.

이유:

- MVP의 핵심 약속은 `5개 버튼 중 하나를 누르면 3초 안에 좋은 목소리로 한마디가 나온다`이다.
- 카테고리 선택 뒤 목록 선택을 한 번 더 요구하면 즉시성이 약해진다.
- 지금은 탐색보다 감정적으로 필요한 말을 바로 듣는 경험이 더 중요하다.

유지할 메인 UX:

- 홈의 5개 버튼을 누르면 해당 카테고리의 한마디를 랜덤으로 즉시 재생한다.
- 현재 한마디 카드는 홈 안에서 갱신된다.
- `다시 듣기`와 `다른 한마디`로 사용자가 현재 카테고리 안에서 가볍게 조정한다.

보조 UX 후보:

- 재생 카드 안에 `이 카테고리 전체 보기` 버튼을 둘 수 있다.
- 누르면 해당 카테고리의 5개 한마디를 직접 선택해 들을 수 있다.
- 이 기능은 v0.1 필수가 아니라, 25개 오디오 연결과 첫 사용 테스트 이후 v0.2 후보로 판단한다.
- 화면 방식은 새 메인 페이지보다 하단 시트나 가벼운 리스트 패널이 더 적합하다.

## 첫 화면 정보 구조

1. 브랜드 영역
   - 앱명: `칭찬해줘`
   - 서브카피: `지금 듣고 싶은 그 말`
   - 안내 문구: `버튼을 누르면 짧은 음성이 재생돼요.`

2. 핵심 CTA 영역
   - `응원해줘!`
   - `진정시켜줘!`
   - `위로해줘!`
   - `괜찮다고 해줘!`
   - `잘했다고 해줘!`

3. 현재 한마디 카드
   - 선택 전: 빈 상태 안내
   - 선택 후: 카테고리명, displayText, 재생 상태, fallback 메시지
   - 액션: 재생/일시정지, 다시 듣기, 다른 한마디
   - v0.2 후보 액션: 이 카테고리 전체 보기

4. 마지막 들은 한마디
   - 최근 들은 문장 3개까지 표시
   - 앱이 기록을 강요하지 않고 조용히 남겨주는 톤

## 버튼 설계

버튼은 이 앱의 가장 중요한 CTA다.

원칙:

- 버튼은 작게 숨기지 않는다.
- 5개 버튼이 첫 화면에서 가장 먼저 보이게 한다.
- 분석형 카테고리명이 아니라 사용자가 앱에게 말하는 요청형 문장으로 둔다.
- 오른쪽에 재생 아이콘을 둬서 "누르면 소리가 난다"를 즉시 알린다.

버튼별 의미:

| 버튼 | 사용 순간 | 색감 |
| --- | --- | --- |
| `응원해줘!` | 다시 움직이고 싶을 때 | coral |
| `진정시켜줘!` | 머리가 복잡하고 불안할 때 | blue |
| `위로해줘!` | 마음이 닳고 지쳤을 때 | violet |
| `괜찮다고 해줘!` | 실수와 자책이 커질 때 | green |
| `잘했다고 해줘!` | 수고를 알아줬으면 할 때 | amber |

## 카드 설계

### 초기 상태

아직 아무 버튼도 누르지 않은 상태에서는 빈 카드가 보인다.

목표:

- 강요하지 않는다.
- "위 버튼을 누르면 여기가 바뀐다"를 알려준다.
- 기록/분석보다 즉시 듣기를 먼저 안내한다.

### 선택 상태

버튼을 누르면 현재 한마디 카드가 갱신된다.

표시:

- 선택된 카테고리명
- 선택된 `displayText`
- 재생 상태
- 재생/다시 듣기/다른 한마디

### 재생 중 상태

오디오가 재생되면 상태 라벨이 `재생 중`으로 바뀐다.
재생 버튼은 `일시정지`로 바뀐다.

### fallback 상태

오디오 파일이 없거나 실패하면 텍스트 fallback 메시지를 보여준다.

현재 메시지:

> 아직 음성 파일이 준비되지 않아 텍스트로 먼저 보여드릴게요.

MVP에서 실제 오디오 파일이 붙기 전까지 이 상태는 정상 상태다.

## 비주얼 시스템

### 배경

- 크림/피치 계열의 따뜻한 배경
- 장식성 블러 원형이나 과한 그라데이션 장식은 피한다.
- 앱이 정신없이 보이지 않게 면 배경 중심으로 둔다.

### 컬러

전체는 크림/피치 기반이고, 버튼별로 5개의 파스텔 포인트를 사용한다.

- coral: 응원
- blue: 진정
- violet: 위로
- green: 괜찮음
- amber: 인정/칭찬

### 타이포그래피

- 앱명은 크고 선명하게
- 버튼명은 크게
- 버튼 설명은 작고 낮은 대비
- 한마디 문장은 카드 안에서 가장 강하게
- 설명 문구는 짧게

### 여백

- 모바일 기준 넉넉한 상하 여백
- 버튼 간 간격은 충분히 둔다.
- 카드 내부는 답답하지 않게 한다.

## 하지 않을 것

- 랜딩 페이지형 hero
- 복잡한 온보딩
- 별도 재생 화면
- 실제 알림 권한 요청
- 결제/광고/계정
- 서버/AI 상담
- 문장과 오디오의 컴포넌트 하드코딩
- 영화/드라마 원본 음성 또는 성대모사

## 현재 남은 디자인 과제

1. 실제 오디오 파일이 붙은 상태로 재생/일시정지 UX를 다시 확인한다.
2. 25개 문장을 실제 음성으로 들었을 때 카드 길이와 줄바꿈을 QA한다.
3. fallback이 아닌 정상 재생 상태의 시각적 피드백을 조금 더 다듬는다.
4. 오디오가 확정되면 첫 사용 테스트에서 버튼명/문장/음성 톤을 같이 검증한다.
5. 25개 오디오 연결 후 `이 카테고리 전체 보기`가 반복 사용에 도움 되는지 v0.2 후보로 평가한다.
