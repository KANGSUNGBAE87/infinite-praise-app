## 결론

무한칭찬앱은 챌린저스, 마이루틴, 루티너리, 알라미, 터닝, Lock In류와 정면 경쟁하면 안 된다.
이 앱들은 대부분 사용자를 "하게 만드는" 제품이다. 무한칭찬앱의 좋은 포지션은 사용자가 이미 버티고, 시작하고, 참았는데도 스스로를 못 알아볼 때 2탭 안에 "나를 인정하게 만드는" 제품이다.

추천 방향은 **루틴 앱이 아니라 루틴/집중/기상 뒤에 붙는 자기인정 레이어**다.

## 현재 무한칭찬앱 기준

- 핵심 경험: 상황 선택 + 감정 선택 + 로컬 칭찬 문장 + 작은 TTS.
- 칭찬의 주체는 AI가 아니라 사용자.
- MVP는 LLM, 서버, API key 없이 오프라인 동작.
- 로컬 기록으로 주간/월간 리포트를 만든다.
- 강제성, 경쟁, 금전 보상, 생산성 평가를 메인으로 두지 않는다.

## 무한칭찬앱의 장점

- 진입 마찰이 낮다. 루틴을 만들거나 돈을 걸거나 앱 권한을 주지 않아도 시작된다.
- 사용자가 지친 상태에서도 쓸 수 있다. 실패를 추궁하지 않고 현재 상태를 인정한다.
- 앱인토스 MVP에 맞다. 로컬 템플릿/TTS 중심이라 서버, 결제, 차단 권한 없이 만들 수 있다.
- 음성은 차별점이 될 수 있다. 루티너리가 "다음 할 일 안내"에 음성을 쓰는 반면, 무한칭찬앱은 자기인정 자체를 들려준다.
- 리포트가 근거를 만든다. 챌린저스의 인증/보상 감각을 돈 없이 "이번 주 나는 이런 상황을 통과했다"로 바꿀 수 있다.

## 약점과 리스크

- 외부 강제력이 약하다. 챌린저스/알라미/터닝처럼 사용자를 강제로 움직이게 하지는 않는다.
- 신선도가 빨리 떨어질 수 있다. 칭찬 문장이 반복되면 "랜덤 위로 문구 앱"으로 느껴진다.
- 초기 사용 이유가 불명확할 수 있다. "왜 매일 켜야 하지?"에 답해야 한다.
- 생산성 앱 사용자에게는 너무 부드러워 보일 수 있다.
- 수익화가 즉시 명확하지 않다. 광고/IAP를 넣으면 신뢰와 감정 톤을 해칠 수 있다.

## 추천 제품 포지션

한 줄:

> 오늘도 뭔가 해낸 것 같지 않은 사람에게, 2탭 안에 자기인정의 목소리를 들려주는 앱.

좁은 초기 타깃:

- 갓생/루틴 앱을 깔아봤지만 실패감이 더 커진 20-30대.
- 공부, 취준, 직장, 프리랜스 생활 중 "오늘 아무것도 못 했다"는 감각이 자주 오는 사람.
- ADHD, 번아웃, 무기력 같은 표현에 공감하지만 치료 앱은 부담스러운 사람.

하지 말아야 할 포지션:

- 완전한 습관 관리 앱.
- 앱 차단/스크린타임 앱.
- 돈 걸고 인증하는 챌린지 앱.
- AI 코칭/상담 앱.

## 구현 고도화 방안

### MVP에 바로 반영할 만한 것

1. **칭찬 강도 선택**
   - 마이루틴의 에너지별 난이도에서 착안.
   - 기본 2탭은 유지하고, 결과 화면에서 `짧게`, `조금 더`, `깊게`를 제공한다.
   - 데이터 모델에 `depth: "short" | "normal" | "deep"`을 추가할 수 있다.

2. **다시 듣기 / 저장 / 별로예요 신호**
   - Lock In류의 progress/reward를 감정 신호로 바꾼다.
   - `replayed`, `saved`, `dismissed`를 저장해 `PraiseSelector`의 가중치로 쓴다.
   - 문장 반복 방지보다 더 좋은 개인화 신호가 된다.

3. **칭찬 영수증**
   - 챌린저스의 인증 감각을 압박 없이 변형한다.
   - 예: "이번 주 `버텼다` 4번", "밤에 나를 챙긴 날 3번".
   - 점수 대신 "흔적"으로 표현한다.

4. **20초 칭찬 루틴**
   - 루티너리의 음성 안내를 행동 지시가 아니라 자기인정 루틴으로 바꾼다.
   - 흐름: 칭찬 문장 재생 → 1초 쉼 → 짧은 호흡/물 한 모금/오늘 내려놓기 문장.
   - 사용자가 원할 때만 실행한다. 메인 2탭 흐름을 방해하지 않는다.

5. **폰/집중 관련 상황 보강**
   - 터닝/Lock In류 수요를 흡수하되 앱 잠금은 하지 않는다.
   - 기존 상황 중 `참았다`, `시작했다`, `끝냈다`에 "폰을 내려놓음", "딴짓을 멈춤" 예시 문장을 넣는다.
   - 필요하면 V1.1에서 새 상황 `돌아왔다`를 추가한다. 의미: 산만함에서 다시 돌아옴.

### V1.1에서 검토할 것

- 부드러운 알림: 하루 1회, 사용자가 고른 시간만. 실패 추궁 문구 금지.
- 홈/위젯: "오늘 나 칭찬하기" 원탭 진입. 앱인토스 제약이 있으면 독립 앱 V2에서.
- 7일 자기인정 카드: streak가 아니라 "나를 알아본 날" 기록.
- 공유 카드: 남에게 자랑하기보다 나에게 저장하기 우선.
- 상황별 음성 톤: 차분함, 단단함, 다정함. 과장된 응원 금지.

### V2에서 검토할 것

- 독립 앱에서 Screen Time / Focus / HealthKit / Health Connect 연동.
- 알라미식 "기상 후 칭찬", 터닝식 "집중 후 칭찬" 자동 트리거.
- LLM은 코치가 아니라 문장 변주/월간 회고 보조로만 도입.
- 팀/B2B는 "성과 챌린지"가 아니라 "번아웃 방지 자기인정 캠페인"으로 별도 실험.

## 데이터 모델 보강 초안

```typescript
interface PraiseEvent {
  id: string;
  date: string;
  situation: PraiseSituation;
  mood: PraiseMood;
  messageId: string;
  message: string;
  depth: "short" | "normal" | "deep";
  source: "manual" | "reminder" | "focus_after" | "wake_after";
  reaction?: "saved" | "replayed" | "dismissed";
  voiceEnabled: boolean;
  createdAt: number;
}
```

## 우선순위

1. 현재 계획은 유지한다. 핵심은 2탭 칭찬 + TTS + 로컬 기록이다.
2. MVP Task 2, 3에 `depth`, `reaction`, `receipt copy`를 추가한다.
3. Task 5 결과 화면에 `다시 듣기`, `저장`, `조금 더 듣기`를 넣는다.
4. Task 6 리포트를 "성과 분석"이 아니라 "자기인정 영수증" 톤으로 쓴다.
5. 앱 잠금, 돈 걸기, AI 코칭, 완전한 루틴 플래너는 MVP에서 제외한다.

## 성공 지표

- 첫 칭찬까지 10초 이하.
- 칭찬 재생 시작 2초 이하.
- D1 재사용률.
- D7에 최소 3회 이상 자기인정 이벤트를 만든 사용자 비율.
- `다시 듣기` 또는 `저장` 비율.
- 리포트 열람률.
- 알림 끄기 비율.
- 같은 문장/톤에 대한 `dismissed` 비율.
