## 0. 시장조사 근거 (왜 이 방향인가)

| 발견 | 출처 | 기획에 반영 |
|------|------|------------|
| Finch 성공 = 펫과의 **감정적 유대**가 리텐션 엔진. 처벌·수치심 0 | 앱 리뷰/UX 분석 | 동반자 캐릭터 + 긍정만 |
| 앱 가치는 **day 5**에 드러남 → 그 전에 삭제 | mood app 이탈 분석 | day 2~5 후크 집중 설계 |
| 자유 저널링은 지속 어려움, 이모지 체크인이 생존 | journaling app 이탈 | 입력은 한 줄 + 캐릭터 반응 |
| "AI가 기록 해석"은 Rosebud·Stoic이 이미 점유 | AI 저널링 트렌드 | 단순 AI 해석 대신 **캐릭터+서사**로 차별화 |
| 무료 미끼·프라이버시 침해가 불만 1순위 | 리뷰 불만 | 코어 무료 + 로컬 저장 명시 |

**빈틈(시장 공백):** "감정적 유대(Finch) × 데이터 기반 자기발견(우리 코어)"를 합친 앱이 없다.
Finch 펫은 내 행동 데이터를 근거로 "넌 이런 사람이야"라고 발견시켜주지 않는다.

## 1. 한 줄 비전

> 나를 지켜보고 기억하는 작은 존재가, 내 실제 기록을 근거로
> "너 생각보다 괜찮은 사람이야"를 매일 증명해준다.

---

## 2. 핵심 컨셉: 관찰자 동반자

펫(내가 돌보는 대상)이 아니라 **관찰자(나를 기억해주는 존재)**.

- 캐릭터는 내가 먹이고 씻기는 대상이 아니다. **내 하루를 듣고, 기억하고, 의미를 찾아주는 존재**다.
- 캐릭터의 성장 = 레벨업이 아니라 **"나에 대한 이해의 깊이"**가 깊어지는 것.
- 사용자가 캐릭터 이름을 직접 짓는다(애착 형성, Finch 검증된 장치).

**3대 약점 → 해결 매핑:**

| 약점(사용자 지적) | 동반자가 해결하는 방식 |
|------------------|----------------------|
| 다시 열 이유 약함 | 캐릭터가 매일 나를 기다리고, **어제 기록과 연결된 한마디**를 건넴 |
| 너무 단순/얕음 | 캐릭터가 **기억**하고 **질문**하고 **정체성을 발견** (단순 칭찬 → 관계로) |
| 감정/재미 없음 | 캐릭터 비주얼 진화 + 관계 단계 + 말투/성격 |

---

## 3. 핵심 메커니즘 4가지

### M1. 기억 (Memory) — 깊이의 핵심
캐릭터가 과거 기록을 기억하고 오늘과 연결한다.
- "지난주에 힘들다고 했었는데, 오늘 운동까지 했네. 너 회복하는 중이구나. *(지난 7일 중 5일 기록)*"
- 단순 집계가 아니라 **시간을 가로지르는 연결**이 "나를 안다"는 느낌을 만든다.
- 구현: 최근 N일 기록을 LLM 컨텍스트에 넣어 연결고리 생성.

### M2. 데일리 연결 (Daily Hook) — 리텐션의 핵심
매일 캐릭터의 첫 마디가 **어제와 이어진다**.
- 어제 기록 있음: "어제 그 일 어떻게 됐어?"
- 어제 빈 날: "어제는 좀 쉬었구나. 오늘은 뭐 했어?" (추궁 아님)
- day 2~5 고비를 넘기는 장치. "내 캐릭터가 어제를 기억하고 날 기다린다."

### M3. 관계 단계 (Relationship Stages) — 재미·진척감
캐릭터가 나를 알아가는 정도가 단계로 시각화 (Finch의 uber besties 모델 차용).
- 1단계 "처음 만난 사이" → 2단계 "알아가는 중" → 3단계 "너를 아는 친구" → 4단계 "오랜 동반자"
- 단계가 오를수록 캐릭터의 말이 더 개인적·구체적으로 변함.
- 단계 = 누적 기록일 + 연결 깊이로 결정.

### M4. 정체성 발견 (Identity Reveal) — 페이오프
캐릭터가 데이터에서 "너는 이런 사람"을 발견해 선물처럼 준다. (v1의 3단계 칭찬 계승)
- 주간: 작은 발견 ("너 몰아서 하는 사람 아니야, 꾸준한 사람이야")
- 월말: 큰 리빌 ("이번 달 너의 이야기" — Spotify Wrapped 감성 + 캐릭터 내레이션)
- **모든 발견에 영수증(근거 숫자)이 붙는다** — v1 불변 원칙 유지.

## 4. 리텐션 곡선 설계 (day별)

| 시점 | 캐릭터 행동 | 목적 |
|------|------------|------|
| Day 0 | 첫 만남, 캐릭터 이름 짓기, 첫 기록 | 애착 시작 |
| Day 1 | "어제 처음 만났지. 오늘은 어땠어?" | 재방문 동기 |
| Day 2~5 | 어제와 연결된 한마디 (M2) | **고비 돌파** |
| Day 7 | 첫 주간 발견 + 관계 2단계 진입 | 첫 페이오프 |
| Day 14 | 관계 3단계, 캐릭터 말투 깊어짐 | 깊이 체감 |
| Day 30 | 월말 리빌 "너의 이야기 1권" | 큰 감정 보상 |
| Day 30+ | 빈 날에도 줌아웃 위로 (v1 규칙) | 이탈 방지 |

## 5. 기능 정의

### MVP (앱인토스)
- **F1. 첫 만남 온보딩** — 캐릭터 등장, 이름 짓기, 첫 기록 유도 (3스텝 이내)
- **F2. 오늘의 기록** — 하루 한 줄 + 카테고리 자동 태그 (v1 계승, 입력 ≤ 5초)
- **F3. 캐릭터 반응** — 기록 직후 캐릭터가 **기억 연결 + 근거 칭찬** (M1+M4)
- **F4. 데일리 연결 인사** — 앱 열면 캐릭터가 어제와 이어진 첫 마디 (M2)
- **F5. 관계 단계 표시** — 캐릭터 상태/단계 화면 (M3)
- **F6. 기록 히스토리** — 내 기록 + 스트릭 (v1 계승)
- **F7. 빈 날 처리** — 줌아웃 + 리프레이밍, 날조 금지 (v1 계승)

### V1.5 (앱인토스 강화)
- **F8. 주간 발견** — 캐릭터가 주 1회 패턴 발견
- **F9. 월말 리빌 "너의 이야기"** — 캐릭터 내레이션 + 카테고리 집계 + 정체성 서사
- **F10. 캐릭터 비주얼 진화** — 관계 단계별 외형 변화

### V2 (Google Play 독립 + 확장)
- **F11. HealthKit/걸음수 패시브 보강** — "오늘 8천보나 걸었네" 자동 인지 (어댑터)
- **F12. 캐릭터 커스터마이징** — 외형/말투 선택 (수익화 연결)
- **F13. 푸시 동반자** — 캐릭터가 보내는 하루 1회 안부 알림

### 제외 범위 (v1 계승 + 강화)
- 소셜 비교/랭킹 (자존감 킬러)
- 캐릭터 돌보기 노동(먹이/청소) — 우리는 펫이 아님, 의무감 주지 않음
- 다중 기록/일, 복잡한 습관 트래커
- 무료 미끼(코어 기능 유료화) — 코어는 무료

## 7. 수익화 (미리 설계)

- **코어 무료**: 기록·캐릭터 반응·기본 칭찬 (불만 1순위였던 "무료 미끼" 회피)
- **구독 (월 2,900~3,900원)**:
  - 월말 "너의 이야기" 리빌 풀버전 + 아카이브 영구 보관
  - 캐릭터 커스터마이징 (외형/말투)
  - 패시브 데이터 보강 (V2)
- 앱인토스=앱인토스 IAP, Google Play=Play Billing — `PaymentAdapter`로 추상화
- 광고는 앱인토스 제약 가능성 → 구독 우선, 광고는 Google Play 버전 한정
- 영수증 검증은 백엔드 (LLM 프록시 백엔드와 통합)

## 8. 기술 스택 (v1 결정 계승)

- **플랫폼**: 앱인토스 = React Native + `@apps-in-toss/framework`. 어댑터로 Google Play 이식.
- **LLM**: `claude-haiku-4-5`, **백엔드 프록시 경유** (API key 앱에 미보관 — v1 보안 규칙 유지)
- **캐릭터 애니메이션**: React Native + 경량 애니메이션 (필요 시 React Bits 참고 — `지식저장소/docs/tools/react-bits.md`)
- **저장**: `StorageAdapter` (앱인토스 저장소 ↔ AsyncStorage)
- **기억 시스템**: 최근 N일 기록을 프록시로 전달 → LLM이 연결고리 생성

### LLM 프롬프트 추가 규칙 (v1 + 동반자)
- v1 규칙 전부 유지 (근거 인용 / 변주 / 타인비교 금지 / 날조 금지 / 빈 날 작은 진실)
- **추가**: 캐릭터 페르소나 일관성 (말투·관계 단계 반영), 과거 기록 연결 시 실제 기록만 인용

## 9. 성공 지표

### MVP (출시 후 30일)
- **D5 retention ≥ 40%** (시장조사: day 5가 가치 분기점 → 핵심 KPI)
- D7 retention ≥ 35%
- 캐릭터 이름 짓기 완료율 ≥ 80% (애착 형성 게이트)
- 일일 기록율 ≥ 50%

### V1.5 (출시 후 60일)
- D30 retention ≥ 25%
- 월말 "너의 이야기" 열람률 ≥ 60%
- 관계 3단계 도달률 ≥ 30%

### 실패 신호
- D5 < 30% → 데일리 연결(M2) 약함
- 캐릭터 반응 "건너뛰기" > 30% → LLM 품질/페르소나 문제
- 이름 짓기 이탈 > 30% → 온보딩 마찰

## 10. 리스크

| 리스크 | 영향 | 대응 |
|--------|------|------|
| 캐릭터가 "AI 챗봇"처럼 느껴져 정 안 붙음 | 높음 | 페르소나 일관성 + 비주얼 + 관계 단계 |
| 기억 연결이 어색하거나 틀림 | 높음 | 실제 기록만 인용, 연결 실패 시 단순 칭찬 폴백 |
| 캐릭터 요소로 "게임" 분류돼 앱인토스 심사 이슈 | 중간 | 본질은 저널링/웰니스로 포지셔닝, 게임 메커닉 배제 |
| LLM 비용 증가 (기억=긴 컨텍스트) | 낮음 | 최근 N일만 전달, Haiku 저가 (1건 ≈ 1.5원) |
| 캐릭터 톤이 한국 정서와 안 맞음 | 중간 | 한국어 네이티브 프롬프트 + 사용자 테스트 |
