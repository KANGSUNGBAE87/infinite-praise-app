# 칭찬해줘 MVP 기획안 v0.2

작성일: 2026-06-08
상태: approval-needed
사용한 관점: Superpowers brainstorming + gstack CEO/founder review

## 한 줄 정의

`칭찬해줘`는 사용자가 지금 듣고 싶은 말을 5개 버튼 중 하나로 고르면,
미리 준비된 명대사/명언/밈 감성의 짧은 음성을 즉시 들려주는 앱이다.

## 제품 약속

사용자는 분석받고 싶은 것이 아니다.
지금 자기 멘탈에 필요한 "그 말"을 바로 듣고 싶다.

따라서 MVP의 약속은 다음 하나다.

> 3초 안에, 지금 필요한 한마디를, 좋은 목소리로 들려준다.

## gstack CEO Review Verdict

모드: SELECTIVE EXPANSION + HOLD SCOPE

유지할 것:

- 5개 버튼
- 버튼 탭 즉시 음성 재생
- 5개 카테고리 x 5개 문장
- QuotePack + audioManifest 기반 데이터 로딩
- 로컬 오디오 파일 재생
- 앱 런타임에서 AI/TTS API 호출 없음
- 배우/캐릭터 목소리 흉내 없음
- 실제 영화/드라마 음성 클립 없음

확장할 것:

- QuotePack 데이터 구조
- 문장별 권리/리스크 메타데이터
- ElevenLabs 등으로 미리 생성한 오디오 manifest
- 카테고리별 음성 톤 가이드
- 나중에 알림/팩 확장/유료팩이 붙을 수 있는 구조
- ReminderAdapter stub

줄일 것:

- 실제 알림 권한 요청/예약/탭 처리
- 상황/감정 정교 매칭
- AI 상담/코칭
- 사용자 계정
- 결제/광고 화면
- 저장소/히스토리 중심 UX
- 원작 음성/성대모사/장면 재현

## 앱명

확정 앱명: `칭찬해줘`

이전 후보:

- `멘탈 한마디`
- `듣고 싶은 말`
- `그 말 해줘`

정정:

`멘탈 한마디`는 카테고리와 제품 방향을 설명하는 기획 프레임으로 남기고,
실제 앱명은 사용자 요청 기준에 따라 `칭찬해줘`로 둔다.

이름의 장점:

- 버튼형 요청 앱이라는 정체성이 즉시 보인다.
- 위로, 응원, 진정, 인정, 괜찮음을 "칭찬/지지의 넓은 의미"로 품는다.
- 명대사/명언 음성 앱의 정체성과 맞는다.
- `멘탈 케어`보다 덜 의료적이고, `한마디`보다 더 행동을 유도한다.

서브카피 후보:

- `지금 듣고 싶은 그 말`
- `누르면 나오는 내 편 한마디`
- `흔들릴 때 바로 듣는 한마디`

추천 서브카피:

> 지금 듣고 싶은 그 말

## MVP 버튼 5개

버튼은 사용자가 앱에게 요청하는 문장으로 통일한다.

| 내부 키 | 버튼명 | 사용 순간 | 문장 톤 |
| --- | --- | --- | --- |
| `encourage` | `응원해줘!` | 에너지가 필요하거나 다시 움직이고 싶을 때 | 밝지만 밀어붙이지 않음 |
| `calm` | `진정시켜줘!` | 불안, 과열, 머리 복잡함, 패닉 직전 | 낮고 느리고 짧음 |
| `comfort` | `위로해줘!` | 지침, 외로움, 번아웃, 마음이 닳았을 때 | 곁에 있어주는 느낌 |
| `reassure` | `괜찮다고 해줘!` | 실수, 자책, 무너짐, 실패감 | 문제를 가볍게 만들지 않음 |
| `recognize` | `잘했다고 해줘!` | 인정받고 싶을 때, 버틴 일을 알아줬으면 할 때 | 조용한 칭찬/인정 |

## 콘텐츠 원칙

### 1. 텍스트는 명대사/명언/밈 감성을 핵심으로 삼는다

기본 콘텐츠는 오리지널 자기계발 문구처럼 느껴지면 안 된다.
사용자가 "아, 이 말은 진짜 남는다"라고 느껴야 한다.

허용 소스:

- 영화/드라마 명대사 텍스트 후보
- 고전 명언/격언
- 유명 밈/짤 문구
- 설문에서 반복 확인된 듣고 싶은 말
- 위 후보의 정서를 가져온 오리지널 한국어 문장

MVP 구현 기준:

- 기존 권장후보 25개는 `referencePack`으로 둔다.
- 실제 앱에 기본 탑재되는 데이터는 `releasePack`으로 둔다.
- `releasePack`은 권리 리스크가 낮은 일반 문구, 고전 격언, 직접 재작성한 오리지널 문장을 우선한다.
- 유명 명대사는 감정 구조와 톤을 참고하되, 그대로 출시할지는 별도 검수 후 결정한다.

### 2. 음성은 새로 만든다

사용자 결정:

- 배우/캐릭터 목소리 흉내 금지
- 실제 영화/드라마 음성 클립 탑재 금지
- 특정 장면 음성 재현 금지

따라서 오디오는 다음 중 하나만 허용한다.

- ElevenLabs 등에서 만든 중립 생성 음성
- 라이선스가 확인된 TTS/성우 음성
- 사용자가 로컬에서 직접 준비한 개인용 음성

### 3. 앱에는 "문장 + 오디오"가 같이 들어간다

이 앱의 콘텐츠 단위는 텍스트가 아니라 `말해지는 한마디`다.

좋은 후보라도 음성으로 들었을 때 어색하면 탈락한다.
반대로 평범한 문장도 음성으로 들었을 때 좋으면 살아남을 수 있다.

## QuotePack 구조

MVP는 두 종류의 데이터팩을 구분한다.

### referencePack

- 기획/선별용 후보 모음
- 기존 권장후보 25개를 포함한다.
- 유명 영화/드라마/밈 후보도 들어갈 수 있다.
- 제품 화면에는 기본 노출하지 않는다.
- `riskLevel: high`여도 남겨둘 수 있지만, 출시 후보라는 뜻은 아니다.

### releasePack

- 실제 앱에서 읽는 기본 콘텐츠
- 5개 카테고리
- 카테고리별 5개
- 총 25개
- 권리 리스크가 낮은 문장, 직접 재작성한 문장, 검수된 문장만 들어간다.
- 출시용 항목은 `audioStatus: approved`가 목표다.

MVP 구현은 `releasePack`을 기준으로 한다.

- 5개 카테고리
- 카테고리별 5개
- 총 25개 출시 후보
- 각 항목은 텍스트, 출처, 리스크, 오디오 상태를 가진다.

```ts
type QuoteItem = {
  id: string;
  category: "encourage" | "calm" | "comfort" | "reassure" | "recognize";
  buttonLabel:
    | "응원해줘!"
    | "진정시켜줘!"
    | "위로해줘!"
    | "괜찮다고 해줘!"
    | "잘했다고 해줘!";
  displayText: string;
  spokenText: string;
  sourceTitle?: string;
  sourceType: "movie" | "drama" | "quote" | "proverb" | "meme" | "survey" | "original";
  sourceUrl?: string;
  rightsStatus: "public_domain" | "licensed" | "owned" | "common_phrase" | "unknown";
  riskLevel: "low" | "medium" | "high";
  audioPath?: string;
  audioStatus: "missing" | "generated" | "approved";
  audioPolicy: "neutral_generated_voice" | "licensed_voice" | "user_local_only";
  voiceStyle: "bright" | "calm" | "warm" | "firm" | "proud";
  durationMs?: number;
};
```

## 카테고리별 음성 톤

### 응원해줘!

- 속도: 중간
- 에너지: 밝음
- 금지: "더 해", "참아", "무조건 가능"식 압박
- 좋은 느낌: 같이 출발해주는 목소리

### 진정시켜줘!

- 속도: 느림
- 에너지: 낮음
- 금지: 과한 확신, 명령형, 빠른 템포
- 좋은 느낌: 호흡을 낮춰주는 목소리

### 위로해줘!

- 속도: 느림-중간
- 에너지: 따뜻함
- 금지: 문제를 축소하거나 훈계하기
- 좋은 느낌: 옆에 조용히 앉아주는 목소리

### 괜찮다고 해줘!

- 속도: 중간보다 느림
- 에너지: 단단함
- 금지: "아무 일도 아니야"처럼 현실을 지우기
- 좋은 느낌: 실수를 사람 전체와 분리해주는 목소리

### 잘했다고 해줘!

- 속도: 중간
- 에너지: 낮은 확신
- 금지: 과장된 찬양, 자기계발식 구호
- 좋은 느낌: 네가 한 일을 내가 봤다는 인정

## 화면 흐름

### 첫 화면

요소:

- 앱명 `칭찬해줘`
- 서브카피 `지금 듣고 싶은 그 말`
- 5개 버튼
- 마지막 들은 한마디가 있으면 "다시 듣기"

버튼:

- `응원해줘!`
- `진정시켜줘!`
- `위로해줘!`
- `괜찮다고 해줘!`
- `잘했다고 해줘!`

### 재생 화면

요소:

- 선택한 버튼명
- 현재 한마디 텍스트
- 큰 재생/일시정지 버튼
- 다시 듣기
- 다른 한마디

원칙:

- 사용자가 버튼을 누른 후 3초 안에 음성이 나와야 한다.
- 오디오 파일이 없으면 개발 중에는 fallback 음성으로 대체하되, 출시용 항목은 `audioStatus: approved`만 사용한다.

### 알림

v0.1 구현 범위:

- 실제 알림 기능은 구현하지 않는다.
- `ReminderAdapter`는 향후 확장을 위한 stub만 만든다.
- 화면에는 알림 기능을 핵심 흐름으로 넣지 않는다.

v0.2 이후 후보 흐름:

- 재생 화면에 `알림으로 받기`를 추가한다.
- 사용자가 특정 카테고리에서 `알림으로 받기`를 누른다.
- 시간을 고른다.
- 알림이 도착한다.
- 사용자가 알림을 탭한다.
- 앱이 열리고 해당 카테고리의 한마디를 재생 준비한다.

제외:

- 실제 알림 권한 요청
- 알림 예약
- 알림 탭 처리
- 알림 도착 즉시 자동 음성 재생
- 백그라운드 자동 재생
- 서버 push

## 구현 준비 범위

MVP 구현 시 필요한 구조:

- `quote-pack.v0.1.json`
- `audio-manifest.v0.1.json`
- `AudioPlayer` 어댑터
- `ReminderAdapter` 스텁
- `PlatformAdapter` 스텁
- i18n 준비
- Apps in Toss / Google Play 호환성을 깨지 않는 구조

아직 하지 않을 것:

- 계정
- 결제
- 광고
- 서버
- 실시간 TTS API
- AI 문장 생성
- 팩 구매
- 실제 알림 권한 요청
- 알림 예약
- 알림 탭 처리

## 성공 기준

MVP가 성공하려면 다음을 만족해야 한다.

1. 첫 화면에서 5개 버튼의 의미가 즉시 이해된다.
2. 버튼 탭 후 3초 안에 음성이 들린다.
3. 같은 버튼을 여러 번 눌러도 문장이 너무 반복적으로 느껴지지 않는다.
4. 음성이 브라우저 기본 TTS처럼 허접하게 들리지 않는다.
5. 각 카테고리마다 "왜 이 문장이 여기 들어갔는지" 설명할 수 있다.
6. 권리 리스크가 높은 후보는 제품 데이터에서 분리되어 있다.
7. 문장과 오디오는 화면 컴포넌트에 하드코딩되지 않는다.
8. 오디오 파일이 없거나 재생 실패해도 앱이 깨지지 않고 텍스트를 보여준다.
9. ReminderAdapter는 stub으로 남아 있고 실제 알림 권한 요청은 하지 않는다.

## 다음 의사결정

구현 전에 아래 세 가지를 확정해야 한다.

1. 앱명: `칭찬해줘`
2. 버튼명: `응원해줘! / 진정시켜줘! / 위로해줘! / 괜찮다고 해줘! / 잘했다고 해줘!`
3. QuotePack: 권장 후보 25개는 `referencePack`으로 두고, 실제 앱에는 권리 리스크가 낮은 `releasePack` 25개를 넣는다.

추천:

- 앱명은 `칭찬해줘`로 확정
- 버튼명은 현재 5개로 확정
- 첫 구현은 `5버튼 + releasePack + audioManifest + 로컬 오디오 재생`만 완성
- 알림은 첫 구현에서 제외하고 `ReminderAdapter stub`만 둔다.
- 권장후보 25개는 reference로 삼되, 출시 데이터는 직접 재작성/검수한 문장으로 구성한다.
