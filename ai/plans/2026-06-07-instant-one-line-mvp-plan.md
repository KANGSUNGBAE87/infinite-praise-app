# 기획안: 바로 듣는 한마디 MVP

작성일: 2026-06-07
단계: product design
상태: approval-needed

## 제품 정의

`무한칭찬앱`의 MVP는 더 이상 정교한 감정/상황 매칭 앱이 아니다.

> 생각할 힘이 없을 때, 지금 필요한 짧은 한마디를 바로 듣는 앱

사용자는 긴 선택을 하지 않는다. 이름 입력, 상황 입력, AI 생성, 복잡한 명대사
검색도 하지 않는다. 5개 버튼 중 하나를 누르면 3초 안에 준비된 문장/음성이 나온다.

## 왜 이렇게 줄이는가

이 앱은 "정확한 말"을 찾으려다 기획이 계속 복잡해졌다.

- 상황 x 감정 조합은 개발자 분류표처럼 느껴졌다.
- 드라마/영화 명대사는 감각은 좋지만 권리/수급/매칭이 어렵다.
- 이름 포함 음성은 모든 문장을 다시 음성화해야 해서 유지비가 커진다.
- 알림에서 자동 음성 재생은 플랫폼 제약이 크다.

따라서 MVP는 "최소 선택, 짧은 음성, 미리 준비한 파일"로 간다.

## 5개 카테고리

### 1. 힘내자

용도: 에너지를 조금 올리고 싶을 때.

톤: 밝고 짧게. `넌 최고야`, `할 수 있어` 같은 직접 응원 허용.

문장 후보:

1. 너 지금 잘하고 있어.
2. 한 번만 더 가보자.
3. 괜찮아, 아직 끝난 거 아니야.
4. 넌 생각보다 훨씬 잘 버티고 있어.
5. 오늘의 너, 충분히 멋있어.

### 2. 진정하자

용도: 불안하거나 머리가 시끄러울 때.

톤: 낮고 느리게. 해결보다 호흡.

문장 후보:

1. 괜찮아. 지금은 숨부터 쉬자.
2. 천천히 해도 돼.
3. 지금 당장 다 정하지 않아도 돼.
4. 마음이 급해도, 너는 여기 있어.
5. 잠깐 멈춰도 아무 일 안 나.

### 3. 쉬어도 돼

용도: 지치거나 쉬는 데 죄책감이 들 때.

톤: 허락, 낮은 압력.

문장 후보:

1. 오늘은 여기까지 해도 돼.
2. 쉬는 것도 필요한 일이야.
3. 더 밀지 않아도 돼.
4. 멈춘 게 아니라 숨을 고른 거야.
5. 지금 쉬어도 너는 사라지지 않아.

### 4. 괜찮아

용도: 실수, 자책, 무너진 감정이 있을 때.

톤: 감싸기, 비난 중단.

문장 후보:

1. 그럴 수 있어.
2. 너무 몰아붙이지 않아도 돼.
3. 오늘 하나 틀렸다고 네가 틀린 건 아니야.
4. 무너진 것 같아도, 끝난 건 아니야.
5. 지금의 너도 괜찮아.

### 5. 잘하고 있어

용도: 별 이유 없이 인정받고 싶을 때.

톤: 기본 자기인정, 담백한 칭찬.

문장 후보:

1. 잘하고 있어.
2. 지금도 충분히 해내고 있어.
3. 오늘의 너를 너무 작게 보지 말자.
4. 여기까지 온 것도 일이야.
5. 너는 너대로 잘 가고 있어.

## 명대사/명언 파일 연결 방향

MVP의 25문장은 오리지널 문장으로 시작한다. 드라마/영화 명대사나 명언은
사용자가 준비한 별도 파일로 확장한다.

권장 파일 구조:

```json
{
  "id": "quote-001",
  "enabled": true,
  "category": "진정하자",
  "line": "권리 확인된 문장",
  "audioPath": "assets/audio/quote-001.wav",
  "sourceType": "movie",
  "sourceTitle": "작품명",
  "rightsStatus": "licensed",
  "displayAttribution": "작품명"
}
```

권리 상태:

- `owned`: 직접 만든 문장/음성
- `licensed`: 권리 확보된 명대사/음성
- `public_domain`: 퍼블릭 도메인 명언
- `user_local`: 사용자가 개인적으로 넣은 로컬 파일
- `unknown`: 앱에 노출하지 않음

앱은 `unknown`을 재생하지 않는다.

## 음성 전략

MVP는 실시간 TTS가 아니라 사전 생성 음성을 쓴다.

- 5개 카테고리 x 5문장 = 25개 오디오
- ElevenLabs, Gemini TTS, 직접 녹음 중 하나로 생성
- 앱에는 API key를 넣지 않는다.
- 생성 파일만 앱 assets로 넣는다.
- 문장 변경 시 해당 파일만 다시 만든다.

이 방식의 장점:

- 비용 통제 가능
- 앱 실행 속도 빠름
- 네가 원하는 말투로 직접 검수 가능
- 이름 입력 같은 개인화 때문에 모든 음성을 다시 생성하는 문제를 피함

## 알림 기능

알림은 "정해진 시간에 자동으로 음성을 재생"이 아니라 다음 구조가 안전하다.

1. 사용자가 카테고리를 고른다.
2. 시간을 고른다.
3. 해당 시간에 알림이 온다.
4. 알림 문구는 짧은 텍스트로 온다.
5. 사용자가 누르면 앱이 열리고 해당 카테고리 음성을 바로 재생한다.

예:

- 카테고리: 힘내자
- 시간: 매일 08:30
- 알림 텍스트: "지금 한마디 들을 시간"
- 탭 후 앱에서 `힘내자` 음성 재생

주의:

- 웹/브라우저는 탭이 닫히면 정교한 예약 알림이 어렵다.
- 모바일 앱/앱인토스/네이티브에서는 플랫폼 adapter가 필요하다.
- 자동 음성 재생은 사용자 제스처/플랫폼 정책 때문에 막힐 수 있다.

## 첫 화면 제안

제목:

> 지금 필요한 한마디

버튼:

- 힘내자
- 진정하자
- 쉬어도 돼
- 괜찮아
- 잘하고 있어

동작:

- 버튼을 누르면 바로 재생한다.
- 결과 화면에는 문장, 다시 듣기, 저장, 알림 설정을 보여준다.
- 알림 설정은 선택한 카테고리를 기본값으로 잡는다.

## CEO 리뷰

### 이 방향이 좋은 이유

- 앱의 첫 사용 비용이 거의 0이다.
- 선택지 허접함 문제가 사라진다.
- 사용자가 파일/음성을 직접 준비할 수 있다.
- AI 비용과 API key 문제가 MVP에서 사라진다.
- 음성 품질을 직접 검수할 수 있다.

### 위험

- 너무 단순해서 차별성이 약할 수 있다.
- 명대사 감각은 실제 quote pack이 붙기 전까지 약하다.
- 알림 기능이 없으면 반복 사용 동기가 약할 수 있다.

### 보완

- 첫 MVP는 25문장으로 아주 작게 출시한다.
- 카테고리별 음성 질감을 확실히 다르게 만든다.
- 알림을 빠르게 붙여 "정해진 시간에 한마디 듣는 습관"으로 만든다.
- 이후 명대사/명언 pack은 파일 기반으로 추가한다.

## 권장 다음 단계

1. 이 5개 카테고리와 25문장을 확정한다.
2. 앱 UI를 5버튼 바로 재생 구조로 바꾼다.
3. 오디오 파일이 없어도 브라우저 TTS fallback으로 동작하게 한다.
4. `assets/audio/{category}/{lineId}.wav` 구조를 준비한다.
5. 알림 설정 UI는 먼저 저장만 구현하고, 실제 플랫폼 알림은 adapter로 분리한다.
