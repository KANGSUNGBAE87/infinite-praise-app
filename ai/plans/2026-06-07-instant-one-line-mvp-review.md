# 리뷰: 바로 듣는 한마디 MVP

작성일: 2026-06-07
리뷰 방식: Superpowers brainstorming + gstack CEO/founder review
대상 문서: `ai/plans/2026-06-07-instant-one-line-mvp-plan.md`
상태: review-complete

## 한 줄 판정

이 방향은 맞다. `상황 x 감정 x 명대사 매칭`보다 `5개 버튼 x 짧은 사전 음성`이
MVP로 훨씬 강하다.

다만 현재 기획안은 아직 세 가지가 흐릿하다.

1. 5개 카테고리의 역할 경계
2. 25문장의 품질 기준
3. 알림과 명대사 pack의 MVP 포함 범위

이 세 가지를 고정하지 않으면, 구현 후에도 "허접하다"는 감각이 다시 돌아올 수
있다.

## GStack CEO Review Verdict

모드: SCOPE REDUCTION + HOLD SCOPE

이 제품은 지금 더 똑똑해지면 안 된다. 더 단순해져야 한다.

핵심 가정:

> 사용자는 위로를 분석하고 싶지 않다. 그냥 지금 들을 한마디가 필요하다.

따라서 `바로 듣는 한마디 MVP`의 성공 조건은 정확한 매칭이 아니라 즉시성이다.

- 3초 안에 들린다.
- 선택지는 5개를 넘지 않는다.
- 문장은 1-2문장을 넘지 않는다.
- 음성은 미리 준비되어 끊기지 않는다.
- 사용자는 생각하지 않고 누른다.

## 주요 Findings

### P0. 명대사/명언은 MVP 기본값이 아니다

드라마/영화 명대사와 실제 배우 음성은 제품 감각에는 좋지만, MVP의 기본값으로
두면 권리/수급/검수 때문에 다시 늪에 빠진다.

결정:

- MVP 25문장은 `owned original line`으로 시작한다.
- 명대사/명언은 별도 파일 pack이다.
- `rightsStatus`가 `owned`, `licensed`, `public_domain`, `user_local`인 항목만 노출한다.
- `unknown`은 앱에 노출하지 않는다.

리스크:

- 명대사 감각이 MVP에서 약할 수 있다.

보완:

- 오리지널 문장을 "명대사처럼 짧고 남는 말"로 쓴다.
- 실제 명대사는 나중에 pack으로 붙인다.

### P1. `잘하고 있어`는 카테고리명으로는 약하다

`잘하고 있어`는 좋은 결과 문장이지만, 카테고리명으로는 "내가 왜 이 버튼을
누르는지"가 약하다.

사용자 언어로는 다음이 더 선명하다.

- `칭찬해줘`
- `인정받고 싶어`
- `잘하고 있다고 해줘`

추천:

MVP 버튼명은 `잘하고 있어` 대신 `칭찬해줘`로 둔다. 결과 문장에는
`잘하고 있어.`를 포함한다.

대안:

브랜드 톤을 더 담백하게 유지하고 싶으면 `잘하고 있어`를 그대로 둬도 되지만,
첫 사용자가 버튼 의미를 바로 이해하는 데는 `칭찬해줘`가 더 강하다.

### P1. 5개 카테고리는 감정이 아니라 "필요한 기능"이어야 한다

현재 카테고리는 대체로 맞지만, 각 버튼이 사용자의 어떤 순간을 해결하는지 더
명확해야 한다.

권장 카테고리 정의:

| 버튼 | 사용 순간 | 금지 |
| --- | --- | --- |
| 힘내자 | 에너지를 올리고 싶다 | 죄책감 있는 사람에게 더 하라고 밀기 |
| 진정하자 | 불안/과호흡/머리 복잡 | 미래를 보장하기 |
| 쉬어도 돼 | 지침/죄책감/번아웃 | 쉬었으니 다시 달리자 |
| 괜찮아 | 실수/자책/무너짐 | 문제를 가볍게 취급하기 |
| 칭찬해줘 | 그냥 인정받고 싶음 | 과장된 자기계발식 찬양 |

### P1. 일부 문장은 다시 검수해야 한다

현재 25문장은 MVP용 초안으로는 충분하지만, 음성화 전에 몇 문장은 손봐야 한다.

검수 필요:

- `한 번만 더 가보자.`
  - 문제: 사용자가 지쳤을 때 압박처럼 들릴 수 있다.
  - 수정 후보: `조금만 더 힘을 내도 돼. 아니면 여기서 숨 쉬어도 돼.`

- `잠깐 멈춰도 아무 일 안 나.`
  - 문제: 실제로는 일이 날 수도 있어서 가짜 확신처럼 들린다.
  - 수정 후보: `잠깐 멈춰도 괜찮아. 지금은 속도를 낮추자.`

- `지금 쉬어도 너는 사라지지 않아.`
  - 문제: 표현이 무겁고 약간 섬뜩하다.
  - 수정 후보: `쉬어도 네가 해온 일은 사라지지 않아.`

- `오늘 하나 틀렸다고 네가 틀린 건 아니야.`
  - 유지 가능. 좋은 방향.

### P1. 음성 품질이 제품의 50%다

이 MVP는 텍스트 앱이 아니다. 음성 앱이다.

문장보다 더 중요한 것은 같은 문장을 어떤 숨, 속도, 거리감으로 읽느냐다. 따라서
오디오 파일은 단순 assets가 아니라 제품 데이터다.

권장 오디오 메타데이터:

```json
{
  "id": "calm-001",
  "category": "진정하자",
  "line": "괜찮아. 지금은 숨부터 쉬자.",
  "audioPath": "assets/audio/calm/calm-001.wav",
  "voiceStyle": "low_slow_soft",
  "durationMs": 3200,
  "rightsStatus": "owned"
}
```

### P2. 알림은 MVP에 넣되, 실제 푸시는 adapter 뒤로 숨긴다

알림 아이디어는 좋다. 반복 사용 이유가 생긴다.

하지만 "정해진 시간에 자동 음성 재생"은 MVP 요구사항이 되면 안 된다. 플랫폼
정책과 사용자 제스처 문제로 막힐 가능성이 높다.

MVP 범위:

- 카테고리별 알림 시간 설정 UI
- 설정값 저장
- 웹에서는 "다음 알림 예정" 표시
- 실제 푸시는 platform adapter TODO로 분리

모바일/앱인토스 범위:

- 알림 도착
- 알림 탭 시 앱 열림
- 해당 카테고리 음성 자동 준비
- 가능하면 탭 후 재생, 자동 재생은 플랫폼 허용 범위에서만

### P2. 저장 기능은 앱으로 돌아오는 이유가 약하다

사용자가 저장해도 다시 앱으로 돌아올 이유가 약하다는 지적은 맞다.

저장은 "아카이브"가 아니라 "다음 알림/재생 선호 신호"로 써야 한다.

추천:

- 저장한 문장은 해당 카테고리에서 더 자주 나온다.
- 저장 후 바로 "이 한마디를 언제 다시 들을까?"를 묻는다.
- 알림 설정의 기본값은 방금 저장한 카테고리다.

## 확정 추천 MVP Scope

포함:

- 5개 버튼
- 카테고리별 5문장
- 버튼 누르면 즉시 한마디 재생
- 다시 듣기
- 저장
- 저장 후 알림 시간 설정 UI
- 오디오 파일이 없으면 브라우저 TTS fallback
- 오디오 파일 manifest 구조

제외:

- 이름 입력
- AI 문장 생성
- 실시간 외부 TTS 호출
- 실제 명대사/배우 음성 기본 탑재
- 자동 음성 push
- 사용자 계정/결제/광고 UI

준비만:

- 명대사/명언 pack schema
- 알림 platform adapter
- Apps in Toss / Google Play portability

## 카테고리/문장 개정안

### 힘내자

1. 너 지금 잘하고 있어.
2. 아직 끝난 거 아니야.
3. 오늘의 너, 꽤 멋있어.
4. 여기까지 온 것도 힘이야.
5. 조금만 더 가보자.

### 진정하자

1. 괜찮아. 지금은 숨부터 쉬자.
2. 천천히 해도 돼.
3. 지금 당장 다 정하지 않아도 돼.
4. 마음이 급해도, 너는 여기 있어.
5. 잠깐 멈춰도 괜찮아.

### 쉬어도 돼

1. 오늘은 여기까지 해도 돼.
2. 쉬는 것도 필요한 일이야.
3. 더 밀지 않아도 돼.
4. 멈춘 게 아니라 숨을 고른 거야.
5. 쉬어도 네가 해온 일은 사라지지 않아.

### 괜찮아

1. 그럴 수 있어.
2. 너무 몰아붙이지 않아도 돼.
3. 하나 틀렸다고 네가 틀린 건 아니야.
4. 무너진 것 같아도 끝난 건 아니야.
5. 지금의 너도 괜찮아.

### 칭찬해줘

1. 잘하고 있어.
2. 지금도 충분히 해내고 있어.
3. 오늘의 너를 너무 작게 보지 말자.
4. 여기까지 온 것도 일이야.
5. 너는 너대로 잘 가고 있어.

## 최종 권고

이 기획안은 구현으로 넘어가도 된다. 단, 다음 조건으로만 승인한다.

1. 버튼명은 `잘하고 있어`보다 `칭찬해줘`를 우선 검토한다.
2. 명대사/명언은 MVP 기본 기능에서 제외하고 pack 구조만 준비한다.
3. 알림은 "탭하면 앱에서 음성을 듣는 구조"로 제한한다.
4. 25문장은 음성화 전 최종 검수한다.
5. 오디오 manifest를 제품 데이터로 다룬다.

이 조건을 지키면 MVP는 명확하다.

> 5개 버튼. 25개 한마디. 누르면 바로 듣는다. 저장하면 다시 들을 시간이 생긴다.
