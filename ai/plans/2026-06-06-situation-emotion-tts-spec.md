# 제품 스펙: 무한칭찬앱 상황 x 감정 직접 칭찬 MVP

작성일: 2026-06-06
단계: 제품 스펙
상태: current
이전안: `ai/plans/2026-06-06-product-spec.md` 를 대체

## 제품 비전

사용자가 스스로를 칭찬하는 행동을 직접 만들고, 앱은 그 선택의 흔적을 모아 "생각보다 나는 나를 챙기고 있었다"는 감각을 돌려준다.

## 핵심 전환

이전안은 "기록을 입력하면 LLM이 근거 기반 칭찬을 생성한다"가 중심이었다.
새 MVP는 "사용자가 상황과 감정을 골라 직접 나를 칭찬한다"가 중심이다.

## 설계 원칙

- 칭찬의 주체는 AI가 아니라 사용자다.
- 앱은 판단하거나 평가하지 않고, 사용자의 자기인정 행동을 돕는다.
- 메인 경험은 한 번 눌렀을 때 즉시 반응해야 한다.
- 근거 기반 리포트는 서브 메뉴다. 메인 칭찬 버튼을 방해하지 않는다.
- MVP에서는 LLM, 서버, API key, 백엔드 프록시를 쓰지 않는다.
- 음성은 출시급 성우 녹음이 아니라 작은 TTS로 시작한다.
- 개인화는 LLM 없이 로컬 규칙 엔진과 템플릿으로 구현한다.

## 메인 경험

1. 사용자가 "오늘 나 칭찬하기" 화면에 들어온다.
2. 상황 버튼을 고른다.
3. 감정 톤을 고른다.
4. 앱이 짧은 칭찬 문장, TTS 음성, 가벼운 애니메이션으로 반응한다.
5. 선택 기록은 로컬에 저장된다.

## 상황 버튼

초기 MVP는 7개 상황으로 시작한다.

| 상황 | 의미 | 예시 |
| --- | --- | --- |
| 버텼다 | 힘든 하루를 통과함 | 출근, 마감, 감정 소진 |
| 시작했다 | 작게라도 손댐 | 공부 시작, 운동 시작, 정리 시작 |
| 끝냈다 | 하나를 완료함 | 업무 완료, 집안일 완료 |
| 쉬었다 | 회복을 선택함 | 낮잠, 산책, 휴식 |
| 참았다 | 감정 폭발을 넘김 | 화, 충동, 말실수 |
| 챙겼다 | 기본 돌봄을 함 | 밥, 씻기, 물, 잠 |
| 용기냈다 | 심리적으로 어려운 행동을 함 | 연락, 대화, 거절, 요청 |

## 감정 톤

초기 MVP는 7개 감정으로 시작한다.

| 감정 | 칭찬 톤 |
| --- | --- |
| 지침 | 낮고 부드럽게, 더 요구하지 않음 |
| 불안 | 안정시키고 시작 자체를 인정 |
| 무기력 | 아주 작은 행동의 의미를 인정 |
| 뿌듯함 | 성취감을 더 선명하게 반사 |
| 화남 | 조절과 경계 세우기를 인정 |
| 죄책감 | 쉬어도 된다는 허용을 제공 |
| 평온함 | 과장 없이 잔잔하게 강화 |

## 칭찬 생성 방식

LLM 대신 로컬 템플릿 카탈로그를 사용한다.

입력 신호:

- 상황
- 감정
- 시간대
- 요일
- 최근 7일 선택 패턴
- 같은 상황/감정의 최근 반복 여부
- 연속 사용일

출력 선택:

1. 상황 x 감정 조합에 맞는 후보 문장을 찾는다.
2. 시간대나 반복 패턴이 있으면 가중치를 더한다.
3. 최근 3회 안에 나온 문장은 제외한다.
4. 가장 적합한 문장을 고르고 기록한다.

예시:

| 상황 | 감정 | 칭찬 |
| --- | --- | --- |
| 버텼다 | 지침 | 오늘은 해낸 게 없어 보여도, 버틴 것 자체가 일이었어. |
| 시작했다 | 불안 | 불안한데도 시작했다는 건 이미 방향을 잡은 거야. |
| 끝냈다 | 뿌듯함 | 좋아. 오늘 하나를 끝까지 데려왔네. |
| 쉬었다 | 죄책감 | 쉬는 것도 너를 계속 가게 하는 선택이야. |
| 참았다 | 화남 | 터뜨리지 않고 넘긴 건 약해서가 아니라 조절한 거야. |
| 챙겼다 | 무기력 | 기운 없는 날에 나를 챙긴 건 작지 않아. |

## 음성/TTS 정책

MVP는 작은 TTS로 시작한다.

- `TtsAdapter` 인터페이스를 둔다.
- 1차 구현은 기기/플랫폼 기본 TTS를 사용한다.
- 앱인토스 런타임에서 기본 TTS가 제한되면, macOS `say` 또는 별도 TTS로 만든 짧은 `.m4a` 파일 20-50개를 번들한다.
- 모든 음성은 끄기/켜기 설정을 제공한다.
- 음성 문장은 1-2문장, 5초 내외를 기준으로 한다.
- 초기 음성 톤은 차분한 친구 톤으로 제한한다. 과장된 응원, 설교, 치료적 단정은 피한다.

## 서브 메뉴: 근거 기반 리포트

근거 기반 칭찬은 메인이 아니라 서브 메뉴다.

제공 정보:

- 이번 주 내가 누른 칭찬 횟수
- 자주 선택한 상황
- 자주 선택한 감정
- 시간대별 패턴
- 최근 7일/30일 변화

예시 문장:

- 이번 주에는 `버텼다`를 4번 눌렀어. 요즘은 버티는 날이 많았던 것 같아.
- 지난 7일 동안 `챙겼다`가 늘었어. 작지만 나를 돌보는 행동이 생기고 있어.

이 리포트도 MVP에서는 LLM 없이 템플릿과 집계로 만든다.

## 핸드폰 데이터 기반 개인화

MVP는 권한 없는 데이터와 앱 내부 기록만 사용한다.

권한 없이 가능한 신호:

- 시간대
- 요일
- 앱 내부 선택 기록
- 연속 사용일
- 최근 반복 패턴

V2에서 검토할 권한 기반 신호:

- 걸음 수, 운동, 수면: HealthKit / Health Connect
- 스크린타임/앱 사용량: 권한과 심사 제약이 커서 후순위

LLM 없이 개인화하는 규칙:

- 밤에는 내려놓는 톤을 우선한다.
- 월요일 아침에는 시작/불안 계열 칭찬을 부드럽게 한다.
- 최근 3일 연속 같은 상황을 누르면 "요즘 반복되고 있음"을 반영한다.
- 사용자가 자주 선택한 감정에는 더 많은 문장 변주를 배정한다.
- 사용자가 특정 칭찬을 다시 듣거나 저장하면 해당 유형 가중치를 올린다.

## MVP 기능 범위

포함:

- 상황 버튼 7개
- 감정 톤 7개
- 상황 x 감정 기반 칭찬 문장 카탈로그
- 작은 TTS 재생
- 칭찬 기록 로컬 저장
- 최근 기록 히스토리
- 주간/월간 근거 리포트 서브 메뉴
- 음성 켜기/끄기

제외:

- LLM 호출
- 백엔드 프록시
- 사용자 계정/동기화
- 인앱결제
- 광고
- HealthKit/Health Connect 연동
- Screen Time 연동
- 성우 녹음

## 성공 기준

- 첫 화면에서 칭찬 재생까지 2탭 이내
- 칭찬 반응은 1초 이내 표시
- 음성 재생은 2초 이내 시작
- 사용자가 같은 문장을 3회 연속 듣지 않음
- 오프라인에서도 모든 MVP 기능 동작
- 리포트 문장은 실제 앱 내부 기록만 사용
