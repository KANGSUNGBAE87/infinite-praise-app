## 질문

현재 무한칭찬앱 문구는 칭찬 카탈로그처럼 들린다. 사용자가 말한 새 방향은
"드라마/영화에서 상황에 따라 가장 듣고 싶은 말"이다. 이 방향이 제품에
맞는지, 어떤 구조로 구현해야 하는지 조사한다.

## 핵심 결론

드라마/영화식 위로는 "칭찬"보다 "증언"에 가깝다.

좋은 한마디는 사용자를 평가하지 않는다. 대신 사용자가 지나온 장면을 정확히
봐주고, 그 장면의 의미를 사용자가 받아들일 수 있는 낮은 압력으로 돌려준다.

현재 앱 문구가 허접하게 느껴진 이유:

- 말하는 사람이 실제 장면을 본 것 같지 않다.
- "의미 있어", "잘했어", "할 수 있어"가 결과 평가처럼 들린다.
- 사용자가 처한 감정의 결을 먼저 봐주지 않고 바로 좋은 말로 덮는다.
- 문장이 너무 설명적이라 드라마 대사가 아니라 자기계발 문장처럼 들린다.

## 좋은 드라마식 위로의 7가지 패턴

### 1. 장면 증언

표면 문장: "그거 봤어."

숨은 의미: 아무도 몰라준 작은 행동을 누군가 알아봤다.

앱 적용:

- "오늘을 끝까지 데리고 온 거, 아무 일 아닌 척 넘기지 말자."
- "손댄 순간부터 오늘은 완전히 멈춘 날이 아니었어."

### 2. 허락

표면 문장: "그만해도 돼."

숨은 의미: 지금 더 증명하지 않아도 관계가 끊기지 않는다.

앱 적용:

- "오늘은 더 밀지 않아도 돼."
- "쉬었다고 사라진 건 아니야."

### 3. 동행/연속성

표면 문장: "내일 보자."

숨은 의미: 오늘 실패해도 너는 내 세계에서 빠지지 않는다.

앱 적용:

- "오늘은 여기까지. 내일의 너한테 넘겨도 돼."
- "다 못 끝내도, 너는 계속 이어지고 있어."

### 4. 책임 내려놓기

표면 문장: "네 잘못만은 아니야."

숨은 의미: 사용자가 자기비난으로 모든 것을 떠안지 않아도 된다.

앱 적용:

- "이 하루가 전부 네 책임처럼 느껴져도, 그렇게 단순한 일은 아니야."
- "못한 것만 세면 네가 한 일이 지워져."

### 5. 감정의 품격 부여

표면 문장: "그렇게 느낄 만했어."

숨은 의미: 화, 불안, 죄책감 같은 감정이 부끄러움만은 아니다.

앱 적용:

- "화가 났다는 건 네 경계가 살아 있었다는 뜻이기도 해."
- "불안했는데 손댄 건, 쉬운 일이 아니야."

### 6. 평범한 것의 회복

표면 문장: "밥 먹었네."

숨은 의미: 삶의 기본을 붙잡은 것도 중요한 사건이다.

앱 적용:

- "밥을 먹고, 물을 마시고, 씻은 것. 그게 오늘의 바닥을 붙잡았어."
- "생활을 다시 세우는 일은 늘 작게 시작돼."

### 7. 말하지 않는 말

표면 문장: 구체적인 행동 하나만 말한다.

숨은 의미: 사랑, 인정, 동행을 직접 설명하지 않는다.

앱 적용:

- "문 닫고 와. 오늘은 더 설명 안 해도 돼."
- "앉아. 여기서는 네가 못한 것부터 말하지 않아도 돼."

## 심리/커뮤니케이션 쪽 적용점

### 유효한 지지 문장은 상황별 전술이다

지원 커뮤니케이션 연구는 좋은 지지가 단순한 메시지 내용만이 아니라, 상황에 맞는
표현, 말의 속도/피치/볼륨 같은 전달 전술과도 관련된다고 본다. 따라서 앱도
`message text`만 관리하지 말고 `spoken scene`, `pause`, `tone`을 함께 관리해야
한다.

### 불안 상황에서는 확답보다 인정이 낫다

불안한 사람에게 "괜찮을 거야"식 확답은 짧게 안심시킬 수 있지만 반복적
안심 추구를 강화할 수 있다. 앱의 불안 톤은 미래 보장보다 현재 감정 인정과
작은 선택의 증언이 맞다.

### 자기연민은 칭찬보다 구조가 좋다

Self-compassion의 핵심은 자기친절, 현재 감정의 균형 잡힌 인식, 고립감 완화다.
앱 카피는 사용자를 띄우기보다 다음 순서를 따라야 한다.

1. 지금 장면을 본다.
2. 감정을 과장하거나 지우지 않는다.
3. 사용자가 혼자 이상한 사람이 아니라는 감각을 준다.
4. 더 하라고 시키지 않는다.

## 현재 7상황별 카피 방향

| 상황 | 사용자가 실제로 듣고 싶은 말 | 피해야 할 말 |
| --- | --- | --- |
| 버텼다 | "오늘 살아남은 것 자체를 누가 봐줬으면" | "대단해, 최고야" |
| 시작했다 | "작게 손댄 것도 멈춘 게 아니라고 해줬으면" | "이제 계속하면 돼" |
| 끝냈다 | "이제 내려놔도 된다고 해줬으면" | "다음 것도 할 수 있어" |
| 쉬었다 | "쉰 게 게으름이 아니라고 해줬으면" | "충전했으니 다시 달리자" |
| 참았다 | "삼킨 말과 멈춘 행동을 봐줬으면" | "잘 참았네, 착하다" |
| 챙겼다 | "밥/물/씻기/잠 같은 기본을 사건으로 봐줬으면" | "자기관리를 잘했어" |
| 용기냈다 | "무서웠다는 걸 지우지 않고 인정해줬으면" | "겁내지 마" |

## 구현 제안

다음 구현은 `칭찬 문장 카탈로그`가 아니라 `scene line catalog`로 바꾸는 편이
좋다.

권장 데이터 구조:

```typescript
interface SceneLineCandidate {
  id: string;
  situation: PraiseSituation;
  mood: PraiseMood;
  sceneNeed:
    | "witness"
    | "permission"
    | "continuity"
    | "release_blame"
    | "dignify_emotion"
    | "ordinary_care"
    | "subtext";
  surfaceLine: string;
  hiddenMeaning: string;
  voiceBeat: "whisper" | "steady" | "warm" | "firm" | "rally";
  pauseAfterMs: number;
}
```

카피 생성 규칙:

- 직접 칭찬보다 장면 증언을 먼저 쓴다.
- "너는 어떤 사람이야"보다 "오늘 네가 한 일"을 말한다.
- 문장 수는 1-2문장으로 제한한다.
- "괜찮아질 거야"처럼 미래를 보장하지 않는다.
- "이제 다시 해보자"처럼 즉시 행동을 요구하지 않는다.
- `힘내자` 톤만 rally를 허용한다.
- 드라마/영화 대사는 직접 복제하지 않는다.

## 샘플 방향

아래 문장은 조사 패턴을 바탕으로 새로 작성한 앱용 샘플이다.

- 버텼다/지침: "오늘은 이긴 날이 아니어도 돼. 끝까지 데리고 온 것만 봐도 돼."
- 시작했다/불안: "불안한데 손댔으면, 그건 작은 일이 아니야."
- 끝냈다/평온: "닫은 건 내려놔도 돼. 오늘은 더 붙잡지 않아도 돼."
- 쉬었다/죄책감: "쉰 게 빠진 건 아니야. 돌아올 자리를 만든 거야."
- 참았다/화남: "그 말을 삼킨 순간, 너는 약해진 게 아니야."
- 챙겼다/무기력: "밥을 챙긴 건 생활을 붙잡은 거야. 아무 일 아닌 척하지 말자."
- 용기냈다/힘내자: "무서웠는데 했잖아. 오늘은 그걸 크게 가져가자."

## 다음 단계

1. 기존 `praiseCatalog.ts`를 `sceneNeed` 기반 구조로 바꾼다.
2. 현재 문장 후보를 폐기하거나 별도 legacy로 두고, 상황별 7패턴 x 톤별 최소 2개씩 새로 작성한다.
3. `voiceScript`는 `voiceBeat`를 받아 속도/피치/쉼을 조절하게 바꾼다.
4. 테스트는 금지어보다 `sceneNeed coverage`, `future guarantee 금지`, `advice 금지`, `direct praise 과다 금지`를 본다.
5. 브라우저 QA는 "읽는 문장"이 아니라 "듣는 장면" 기준으로 확인한다.
