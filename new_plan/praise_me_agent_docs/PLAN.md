# PLAN.md

## 1. 문서 목적

이 문서는 `칭찬해줘`의 현재 구현을 안전하게 유지하면서 상품성과 재방문 경험을 개선하기 위한 실행 계획을 정의한다.
작업자는 `GOAL.md`를 먼저 읽고, 현재 사실은 `STATUS.md`, 검증 기준은 `TESTS.md`에서 확인한다.

---

## 2. 현재 기준점

현재 앱은 다음 6단계 local-first 흐름을 가지고 있다.

1. Landing / Target Confirm
2. Praise Pick
3. Rewrite Optional
4. Time Save + Preview
5. Next-day Check-in
6. Result / Interest Slot

현재 실제 알림, 음성, AI, 계정, 광고, 결제, 백엔드는 연결하지 않는다.

---

## 3. 이번 계획의 목표

### 제품 목표

현재의 `한 번 체험하고 끝나는 흐름`을 다음 구조로 개선한다.

> 한 줄을 고른다 → 저장한다 → 다시 돌아와 체크인한다 → 도움이 된 한 줄을 남긴다 → 오늘의 한 줄을 다시 고른다.

### 디자인 목표

- 기존 A2 웜·파스텔 톤을 유지한다.
- 홈과 체크인 화면의 정보 계층을 강화한다.
- 카드가 많아 보여도 사용자가 다음 행동을 즉시 알 수 있게 한다.
- 단순 장식이 아니라 재방문 행동을 유도하는 패널만 사용한다.

---

## 4. 목표 정보 구조

### 첫 방문

```text
Landing
→ 오늘 필요한 결 선택
→ 한 줄 선택
→ 내 말로 수정(선택)
→ 다시 볼 시간 저장
→ 미리보기
```

### 재방문

```text
재방문 홈
→ 오늘의 한 줄 확인
→ 체크인
→ 도움됐어 / 그냥 그랬어 / 오늘은 바꿀래
→ 도움된 문장 보관
→ 새로운 한 줄 선택
```

### 권장 상위 메뉴

MVP에서는 하단 내비게이션을 최대 3개로 제한한다.

- 오늘
- 보관함
- 설정

보관함이 실제 구현 범위에 아직 없다면 `오늘` 단일 화면 중심으로 유지하고 빈 탭을 만들지 않는다.

## 5. 단계별 구현 계획

## Phase 0. 기준 동결 및 안전 확인

목표: 현재 동작을 깨지 않고 변경 가능한 출발점을 만든다.

작업:

- `GOAL.md`, `PLAN.md`, `STATUS.md`, `TESTS.md`를 프로젝트 루트에 둔다.
- 변경 전 `npm test`, `npm run build`를 실행한다.
- 현재 ko/en 주요 카피와 localStorage schema를 기록한다.
- 현재 6개 화면 스크린샷을 360×740, 390×844에서 저장한다.
- 기존 세션로그와 Graphify 상태를 확인한다.

완료 조건:

- 모든 테스트와 빌드 통과
- 기존 화면 캡처 확보
- 변경 대상과 비대상 범위 명확화

## Phase 1. 재방문 홈 설계

목표: 저장 이후 사용자가 다시 돌아올 이유를 한 화면에서 이해하게 한다.

### 홈 필수 패널

1. **오늘의 한 줄 히어로 패널**
   - 가장 큰 카드
   - 현재 저장 문장
   - `오늘 이 한 줄 보기` 또는 `한 줄 바꾸기` CTA
   - 다음 확인 시간

2. **오늘의 결 선택 패널**
   - `잘 버텼어`
   - `잠깐 쉬어`
   - `다시 해보자`
   - `괜찮아`
   - 2×2 타일 또는 가로 스크롤이 아닌 2열 그리드 권장

3. **나를 챙긴 기록 패널**
   - 강제 streak 대신 최근 7일 중 챙긴 날 표시
   - 예: `이번 주 3번 나를 챙겼어요`
   - 실패/결석 경고 금지

4. **보관함 미리보기 패널**
   - 도움이 된 문장이 있을 때만 표시
   - 최대 2개 미리보기
   - `모두 보기` 보조 액션

### 홈에서 제외

- 광고
- 가격
- 결제
- 과도한 통계
- 긴 설정 목록
- 상담·치료 문구

완료 조건:

- 홈 첫 화면에서 오늘 할 행동이 3초 안에 보임
- 주 CTA가 하나만 가장 강하게 보임
- 360px 폭에서 가로 스크롤 없음

---

## Phase 2. 한 줄 선택 경험 개편

목표: 선택 피로를 줄이면서 문장 탐색의 재미를 만든다.

작업:

- 첫 노출은 3문장 유지
- `다른 한 줄 더 보기`로 최대 5문장 노출
- 문장 위에 감정 결 배지 표시
- 선택 카드 전체를 탭 영역으로 사용
- 선택 시 테두리, 그림자, 작은 체크 아이콘으로 상태 표시
- 선택 후 CTA 문구를 `이 한 줄로 계속`처럼 명확하게 유지

데이터 원칙:

- UI 컴포넌트에 문장을 하드코딩하지 않는다.
- i18n과 콘텐츠 카탈로그 경계를 유지한다.
- 특정 영화·드라마·밈 문구를 기본 release 데이터로 넣지 않는다.

완료 조건:

- 3개 카드가 화면에 과도하게 빽빽하지 않음
- 선택 상태가 색상 외에도 형태/아이콘으로 구분됨
- 키보드와 스크린리더로도 선택 가능

---

## Phase 3. 수정·시간 저장·미리보기 개선

목표: 사용자가 자신의 문장으로 바꾸는 흐름을 부담 없이 만든다.

### 수정 화면

- 입력창은 단일 대형 카드로 구성
- 원문과 수정문을 동시에 복잡하게 보여주지 않음
- 글자 수 안내 제공
- caution/blocked 안내는 입력창 아래 한 곳에만 표시
- 자유 입력 원문은 analytics에 포함하지 않음

### 시간 저장 화면

현재 MVP에서는 실제 알림이 없으므로 과장된 `알림 예약 완료` 표현을 사용하지 않는다.

권장 문구:

- `다시 보고 싶은 시간을 정해보세요`
- `현재 MVP에서는 앱 안에서 미리보기로 저장돼요`

시간 입력:

- native `<input type="time">` 유지
- 기본값 21:30
- 선택된 시간을 자연어로 다시 표시

완료 조건:

- caution은 저장 가능, blocked는 저장 불가
- 잘못된 localStorage 복구 유지
- 실제 알림이 가는 것처럼 오해시키지 않음

---

## Phase 4. 다음 날 체크인 개편

목표: 재방문을 평가가 아니라 자기 이해로 느끼게 한다.

기존 `유지 / 내 말로 수정 / 오늘은 건너뛰기`를 다음처럼 바꾸는 것을 우선 검토한다.

- `도움됐어`
- `그냥 그랬어`
- `오늘은 바꿀래`

### 상태별 결과

- 도움됐어
  - 보관함 저장 제안
  - 오늘도 같은 결을 유지할지 선택
- 그냥 그랬어
  - 다른 결 4개 추천
  - 실패처럼 표현하지 않음
- 오늘은 바꿀래
  - 한 줄 선택 화면으로 복귀

완료 조건:

- 세 선택지가 인격 평가처럼 느껴지지 않음
- 결과 화면에 가격·결제·광고 문구 없음
- 사용자가 흐름을 닫거나 다시 선택할 수 있음

---

## Phase 5. 로컬 보관함과 기록

목표: 재방문 보상을 광고나 결제 대신 `내가 남긴 좋은 문장`으로 만든다.

MVP 범위:

- 도움이 된 한 줄 로컬 저장
- 저장일 표시
- 삭제 가능
- 최대 개수 제한은 필요할 때만 도입
- 서버 동기화 없음

주간 기록:

- `연속 7일` 같은 강제 streak 금지
- `최근 7일 중 나를 챙긴 날` 형태 사용
- 미방문일을 실패로 표시하지 않음

완료 조건:

- 보관함이 비어 있을 때 따뜻한 empty state 제공
- 삭제는 되돌리기 또는 확인 UI 제공
- 데이터가 기기 안에 저장된다는 점을 설명

## Phase 6. 시각 시스템 정리

### 색상 토큰 권장안

```css
--pm-bg: #FFF9F5;
--pm-surface: #FFFFFF;
--pm-primary: #7C6CE7;
--pm-primary-strong: #6655D8;
--pm-primary-soft: #EEEAFE;
--pm-peach: #FFE5D6;
--pm-mint: #DFF4EA;
--pm-yellow: #FFF0B8;
--pm-lilac: #E9E0FF;
--pm-text: #2E2B3A;
--pm-text-muted: #747080;
--pm-border: #EEEAF3;
--pm-danger: #D65353;
--pm-warning-bg: #FFF5D8;
```

### 깊이 원칙

강한 3D가 아니라 부드러운 2.5D를 사용한다.

```css
--pm-shadow-card: 0 10px 30px rgba(57, 45, 105, 0.08);
--pm-shadow-raised: 0 14px 36px rgba(74, 59, 140, 0.14);
--pm-shadow-pressed: 0 4px 12px rgba(74, 59, 140, 0.10);
```

- 카드 상단에 1px 내부 하이라이트 허용
- 눌림 상태는 `translateY(1px)` 정도만 사용
- 광택 플라스틱·메탈 질감 금지
- 네온 글로우 금지

### 반경

- 히어로: 28px
- 일반 카드: 22px
- 타일: 18px
- 버튼: 16px 또는 pill

### 타이포

- 큰 제목: 28–32px / 700–800
- 화면 제목: 22–24px / 700
- 카드 제목: 17–18px / 700
- 본문: 15–16px / 400–500
- 보조문구: 13–14px / 400–500

## 7. Definition of Done

작업은 다음 조건을 모두 만족해야 완료다.

- 목표 UX가 구현됨
- 기존 기능이 깨지지 않음
- `npm test` 통과
- `npm run build` 통과
- 360×740, 390×844 시각 QA 통과
- 한국어/영어 주요 문구 누락 없음
- 자유 입력 원문 외부 전송 없음
- 실제 알림·결제·광고·AI를 암시하는 허위 UI 없음
- 변경 파일과 의사결정이 세션로그에 기록됨
- Graphify가 프로젝트 규칙에 따라 갱신됨
